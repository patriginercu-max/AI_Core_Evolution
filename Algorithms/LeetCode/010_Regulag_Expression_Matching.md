# 力扣第 10 题：正则表达式匹配 (Regular Expression Matching)

---------------------------------------------------------
1. 题目挑战点与核心思路
---------------------------------------------------------
* 核心挑战：
    - 星号（*）的“消除”逻辑：星号最难处理的地方在于它可以匹配 0 次。这意味着像 "a*b*" 这样的模式串可以合法地匹配空字符串 ""。
    - 决策分支：遇到 "字符+*" 时，必须决定是直接忽略这个组合（匹配 0 次），还是在当前位匹配的前提下消耗 s 的字符并保留该组合（匹配多次）。
    - 重复子问题：不使用动态规划记录状态会导致递归解法产生大量重复计算。

* 核心思路：
    - 算法选择：动态规划 (DP)。
    - 状态定义：dp[i][j] 表示 s 的前 i 个字符与 p 的前 j 个字符是否匹配。
    - 初始化逻辑：必须预处理 dp[0][j]，以应对模式串开头出现 "a*b*" 这种能匹配空串的情况。

---------------------------------------------------------
2. 代码实现与详细注释
---------------------------------------------------------
class Solution:
    def isMatch(self, s: str, p: str) -> bool:
        m, n = len(s), len(p)
        # dp[i][j] 代表 s 的前 i 个字符和 p 的前 j 个字符是否匹配
        # 维度为 (m+1) * (n+1) 以包含空字符串的情况
        dp = [[False] * (n + 1) for _ in range(m + 1)]
        
        # 基础情况：空串匹配空模式
        dp[0][0] = True
        
        # 初始化第一行：处理 p 中形如 a* 或 a*b* 这种能匹配空串的模式
        for j in range(2, n + 1):
            if p[j-1] == '*':
                # 状态取决于跳过 '字符+*' 组合后的状态，即看 dp[0][j-2]
                dp[0][j] = dp[0][j-2]

        # 填充状态矩阵
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                # 情况 A：当前模式字符 p[j-1] 不是 '*'
                if p[j-1] != '*':
                    # 如果字符匹配（相等或者是 '.'），状态由左上角 dp[i-1][j-1] 决定
                    if p[j-1] == s[i-1] or p[j-1] == '.':
                        dp[i][j] = dp[i-1][j-1]
                
                # 情况 B：当前模式字符 p[j-1] 是 '*'
                else:
                    # 分支 1：匹配 0 次（即“消除” p[j-2] 和 '*'）
                    # 直接看模式串向前回退两个位置的状态
                    if dp[i][j-2]:
                        dp[i][j] = True
                    # 分支 2：匹配 1 次或多次
                    # 前提是 s[i-1] 必须匹配 '*' 前面的字符 p[j-2]
                    elif p[j-2] == s[i-1] or p[j-2] == '.':
                        # 核心点：消耗 s 的一个字符 i-1，但模式串位置 j 不动
                        dp[i][j] = dp[i-1][j]
        
        return dp[m][n]

---------------------------------------------------------
3. 复杂度分析
---------------------------------------------------------
* 时间复杂度：O(m * n)。其中 m 和 n 分别为字符串 s 和模式串 p 的长度。我们需要填满整个二维状态表格。
* 空间复杂度：O(m * n)。我们需要一个 (m+1) * (n+1) 的二维数组来存储所有的子问题结果。

---------------------------------------------------------
4. 关键点总结与避坑
---------------------------------------------------------
* 理解 dp[i-1][j]：这是处理“匹配多次”的关键。当星号匹配了 s[i-1] 后，星号组合并没有消失，所以模式串索引 j 保持不动，继续看 s 的前面部分。
* 第一行初始化：绝对不能漏掉对 dp[0][j] 的处理，否则无法正确识别 "a*" 匹配空串的情况。
* 索引偏移：因为 dp 表的大小是 (m+1)*(n+1)，在取字符串字符时，s[i-1] 才是当前处理的字符，注意不要越界。
* 避坑：不要试图在单层循环里处理星号，一定要识别出星号是与其前一个字符绑定的二元组合。
