# LeetCode 0008. 字符串转换整数 (atoi)

---

### 1. 题目逻辑与核心考点
本题是经典的字符串模拟题，旨在考察对边界条件 (Edge Cases) 的处理能力。其核心逻辑可以概括为以下四个步骤：
* 前导空格处理：跳过字符串开头的全部空格字符。
* 符号判定：检查紧随其后的字符是否为 '+' 或 '-'，并记录正负号。
* 数字解析：读入连续的数字字符直到遇到非数字字符或到达末尾。
* 溢出处理：结果必须落在 32 位有符号整数范围 [-2^31, 2^31 - 1] 内。若超出，则截断至边界值。

---

### 2. 代码实现与详细注释 (Python)
class Solution:
    
    
    def myAtoi(self, s: str) -> int:
        # 1. 移除前导空格
        s = s.lstrip()
        if not s:
            return 0
        
        res, i, sign = 0, 0, 1
        INT_MAX, INT_MIN = 2**31 - 1, -2**31
        
        # 2. 处理符号位
        if s[0] == '-':
            sign = -1
            i += 1
        elif s[0] == '+':
            i += 1
            
        # 3. 逐位转换并进行溢出预判
        while i < len(s) and s[i].isdigit():
            digit = int(s[i])
            
            # 核心难点：溢出检查
            # 如果当前值已经大于 INT_MAX // 10，或者等于该值且下一位数字大于 7
            if res > INT_MAX // 10 or (res == INT_MAX // 10 and digit > 7):
                return INT_MAX if sign == 1 else INT_MIN
            
            res = res * 10 + digit
            i += 1
            
        return sign * res

---

### 3. 复杂度分析
* 时间复杂度：O(n)
    * 算法仅对字符串进行了一次线性扫描（去空格、读符号、转换数字），n 为输入字符串的长度。
* 空间复杂度：O(1)
    * 除了存储结果和索引的几个变量外，没有使用任何与输入规模相关的额外数据结构。

---

### 4. 常见陷阱与总结
* 空输入处理：在调用 lstrip() 后，如果字符串变为空（例如原字符串为 "  "），必须立即返回 0。
* 非数字字符截断：转换只发生在连续的数字部分。例如 "4193 with words" 返回 4193；而 "words and 987" 因为开头不是数字或符号，返回 0。
* 多重符号判定：像 "+-12" 这种输入，在处理完第一个 + 后，由于下一个字符 - 不是数字，循环不会执行，返回 0。
* 溢出判断的数学逻辑：使用 res > INT_MAX / 10 的预判方式是面试中的标准解法，能有效避免在不支持大数运算的语言中出现溢出崩溃。
