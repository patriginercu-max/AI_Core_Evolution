# 力扣第 7 题：整数反转 (Reverse Integer) 深度解构

---

### Ⅰ. 核心挑战点 (Challenge Points)

1. 数值溢出检查：题目要求环境只能存储 32 位有符号整数。反转后的结果若超出 $[-2^{31}, 2^{31} - 1]$ 范围，必须立即返回 0。
2. 负数取余差异：在 Python 中，负数取余（如 `-123 % 10`）结果为 `7` 而非 `-3`。为避免逻辑混乱，最稳妥的方法是处理绝对值。
3. 数学模拟法：不依赖字符串反转，利用数学运算 `x % 10` 和 `x // 10` 逐位提取，考察基本的算法底层逻辑。

---

### Ⅱ. 核心代码实现 (Code Implementation)

class Solution:
    def reverse(self, x):
        # 1. 确定符号并取绝对值
        # Python 的负数取余和整除逻辑与 C++/Java 不同，取绝对值处理最安全
        sign = 1 if x >= 0 else -1
        x = abs(x)
        res = 0
        
        # 2. 数学模拟反转过程
        while x != 0:
            # 弹出末尾数字
            pop = x % 10
            # 将数字推入结果（整体左移一位）
            res = res * 10 + pop
            # 丢弃已处理的末尾数字
            x //= 10
            
        # 3. 补回符号
        res *= sign
        
        # 4. 溢出检查：32 位有符号整数范围判断
        # -2^31 = -2147483648, 2^31 - 1 = 2147483647
        if res < -2**31 or res > 2**31 - 1:
            return 0
            
        return res

---

### Ⅲ. 核心逻辑拆解 (Key Logic)

1. 数学反转公式：
   `res = res * 10 + pop`。每一次循环，旧的结果都会乘以 10（相当于在十进制下整体向左移动一位），然后腾出个位数位置放入新弹出的数字。

2. 溢出边界值：
   $2^{31} - 1$ 的值是 `2147483647`。如果反转后的结果 `res` 超过这个数，或者小于 `-2147483648`，题目明确要求返回 `0`。

3. 为什么不用字符串反转 `str(x)[::-1]`？
   - 字符串法处理负数号非常麻烦。
   - 字符串法在面试中通常会被追问：“如果不允许将整数转为字符串，你怎么做？”
   - 数学法的时间效率更高，且更符合算法面试的意图。



---

### Ⅳ. 复杂度与建议 (Optimization & Tips)

1. 复杂度分析：时间 O(log(n))，其中 n 是数值的大小（循环次数等于数字的位数）；空间 O(1)，只使用了常数个变量。
2. 语言差异点：在 C++ 或 Java 中，你可以通过 `(res * 10 + pop) / 10 != res` 来快速判断是否发生溢出，但 Python 会自动处理大整数，所以必须手动与边界值对比。
3. 练习建议：这道题是“数值处理”类的经典入门，掌握了它，对于后面第 8 题“字符串转换整数 (atoi)”会有很大帮助。
