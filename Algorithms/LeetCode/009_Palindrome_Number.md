# LeetCode 009. 回文数 (Palindrome Number)

### 1. 题目核心挑战与解题思路

**LeetCode 009. 回文数 (Palindrome Number)**

* **核心挑战：**
    1. **不使用字符串：** 进阶要求不使用 str(x)，考验纯数学逻辑和进制操作。
    2. **内存与溢出：** 反转全量数字可能超过 32 位整数范围。通过“反转一半”能完美规避此问题，因为反转后的数值永远不会超过原数。
    3. **边界过滤：** 负数（如 -121）和末尾为 0 的非零数（如 10）在数学逻辑上必须前置剔除。

* **解题思路：**
    1. **数字切分：** 利用 % 10 摘取末位，利用 // 10 缩减原数。
    2. **中点停止：** 当原数 x 不再大于反转数 reverted_number 时，说明已经处理到了中点。
    3. **奇偶兼容：** 偶数位直接比（12==12）；奇数位去掉反转数的个位再比（12==123//10）。

---

### 2. 代码实现与关键注释

class Solution:
    
    
    def isPalindrome(self, x: int) -> bool:
        # 第一步：排除负数和末尾为 0 的非零数
        if x < 0 or (x % 10 == 0 and x != 0):
            return False

        reverted_number = 0
        # 第二步：反转数字的一半
        # 示例 12321: 
        # 第一次: rev=1, x=1232
        # 第二次: rev=12, x=123
        # 第三次: rev=123, x=12 -> 停止
        while x > reverted_number:
            reverted_number = reverted_number * 10 + x % 10
            x //= 10

        # 第三步：判断是否回文
        # 偶数位：x == reverted_number
        # 奇数位：x == reverted_number // 10 (去掉中间位)
        return x == reverted_number or x == reverted_number // 10

---

### 3. 复杂度分析

* **时间复杂度：** O(log10 n)。每次循环将数字缩小 10 倍，循环次数约为总位数的一半。
* **空间复杂度：** O(1)。仅使用了常量级的额外空间，不随输入规模增长。

---

### 4. 进阶思考与总结

* **数学解法的优雅：** 相比 O(n) 空间的字符串法，数学拆解法达到了最优的 O(1) 空间效率，是面试官最想看到的解法。
* **溢出防范：** 只处理一半数字，意味着变量永远不会溢出，逻辑极度稳健。
* **核心总结：** 掌握取模（%）和整除（//）的配合是解决所有数字位操作题目（如反转整数、二进制转换）的基石。
